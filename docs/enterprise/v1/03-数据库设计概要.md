## 一、新增表

### 1.1 enterprises（企业表）

**ent schema**: `backend/ent/schema/enterprise.go`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | int64 | PK, 自增 | |
| name | string(255) | NOT NULL | 企业名称 |
| short_name | string(100) | | 🆕 企业简称 |
| credit_code | string(50) | | 🆕 统一社会信用代码 |
| address | string(500) | | 🆕 企业地址 |
| scale | string(20) | | 🆕 企业规模（枚举：micro/small/medium/large） |
| industry | string(50) | | 🆕 所属行业（枚举） |
| parent_id | int64 | NOT NULL, DEFAULT 0 | 父企业 ID，0=顶级 |
| status | string(20) | NOT NULL, DEFAULT "active" | active / disabled |
| contact_name | string(100) | | 联系人姓名 |
| contact_phone | string(50) | | 联系人手机号 |
| contact_email | string(255) | | 联系人邮箱 |
| notes | text | | 🆕 备注 |
| balance | decimal(20,8) | NOT NULL, DEFAULT 0 | 🆕 **企业余额（独立资金池）** |
| total_recharged | decimal(20,8) | NOT NULL, DEFAULT 0 | 🆕 **企业累计充值** |
| admin_user_id | int64 | NOT NULL | 指向 users.id，企业管理员的 users 记录 |
| created_at | timestamptz | | |
| updated_at | timestamptz | | |
| deleted_at | timestamptz | | 软删除 |

**索引**:
- `name` — 唯一（WHERE deleted_at IS NULL）
- `parent_id`
- `status`
- `admin_user_id`

**Edge**:
- `parent` (Enterprise → Enterprise, 自引用)
- `children` (Enterprise → Enterprise, 自引用)
- `admin` (Enterprise → User)
- `members` (Enterprise → EnterpriseMember)
- `subscriptions` (Enterprise → EnterpriseSubscription) 🆕

### 1.2 enterprise_members（企业成员表）

**ent schema**: `backend/ent/schema/enterprise_member.go`

**本期约束**: 一人一企业（1:1），一个 user_id 最多一条 active 记录

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | int64 | PK, 自增 | |
| enterprise_id | int64 | NOT NULL, FK→enterprises | 所属企业 |
| user_id | int64 | NOT NULL, FK→users | 关联用户 |
| role | string(20) | NOT NULL, DEFAULT "enterprise_member" | enterprise_admin / enterprise_member |
| status | string(20) | NOT NULL, DEFAULT "active" | active / unbound |
| department_id | int64 | FK→departments | 🆕 所属部门 |
| concurrency | int | DEFAULT 0 | 🆕 成员级并发上限（0=无限） |
| rpm_limit | int | DEFAULT 0 | 🆕 成员级 RPM 上限（0=无限） |
| notes | text | | 🆕 备注 |
| joined_at | timestamptz | NOT NULL, DEFAULT NOW() | 加入时间 |
| unbound_at | timestamptz | | 解绑时间 |
| created_at | timestamptz | | |
| updated_at | timestamptz | | |
| deleted_at | timestamptz | | 软删除 |

**索引**:
- `(enterprise_id, user_id)` — 唯一（WHERE deleted_at IS NULL）— 防同一企业重复加入
- `user_id` — **部分唯一索引**（WHERE status='active' AND deleted_at IS NULL）— **本期 1:1 约束**
- `enterprise_id`
- `role`
- `(enterprise_id, role)` — 查企业管理员/成员
- `department_id` 🆕

> ⚠️ **TODO: 未来 M:N 扩展**：删除 `user_id` 上的唯一索引即可支持一个用户同时属于多个企业。`assigned_to` 已指向 `enterprise_members.id`，无需改动 Key 表。

**Edge**:
- `enterprise` (EnterpriseMember → Enterprise)
- `user` (EnterpriseMember → User)
- `department` (EnterpriseMember → Department) 🆕
- `assigned_keys` (EnterpriseMember ← APIKey)

### 1.3 enterprise_subscriptions（企业套餐表）🆕

**ent schema**: `backend/ent/schema/enterprise_subscription.go`

> **目的**: 企业独立套餐池，与企业管理员个人套餐完全隔离。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | int64 | PK, 自增 | |
| enterprise_id | int64 | NOT NULL, FK→enterprises | 所属企业 |
| group_id | int64 | NOT NULL, FK→groups | 分组 |
| plan_id | int64 | NOT NULL, FK→subscription_plans | 套餐计划 |
| starts_at | timestamptz | NOT NULL | 生效时间 |
| expires_at | timestamptz | | 过期时间 |
| status | string(20) | NOT NULL, DEFAULT "active" | active / expired / suspended |
| daily_usage_usd | decimal(20,10) | DEFAULT 0 | 当日用量 |
| weekly_usage_usd | decimal(20,10) | DEFAULT 0 | 当周用量 |
| monthly_usage_usd | decimal(20,10) | DEFAULT 0 | 当月用量 |
| created_at | timestamptz | NOT NULL, DEFAULT NOW() | |
| updated_at | timestamptz | | |

**索引**:
- `(enterprise_id, status)` — 查企业有效套餐
- `(enterprise_id, group_id)` — 查企业某分组套餐

**Edge**:
- `enterprise` (EnterpriseSubscription → Enterprise)
- `group` (EnterpriseSubscription → Group)
- `plan` (EnterpriseSubscription → SubscriptionPlan)

## 二、修改现有表

### 2.1 api_keys 表变更

| 变更 | 字段 | 类型 | 说明 |
|:--:|------|------|------|
| 🆕 新增 | assigned_to | int64, FK→enterprise_members | NULL=未分配, 有值=企业分配的成员 (`enterprise_members.id`) |
| 🆕 新增 | usage_purpose | string(200) | 用途说明 |
| 🆕 新增 | bound_tool | string(50) | 绑定工具：cursor/trae/claude_code/codex/opencode/pixso/other |
| ⚠️ 废弃 | group_id | int64 | 保留字段，新逻辑从 api_key_groups 读取 |

**新增索引**:
- `assigned_to`
- `(assigned_to, status)` — 查成员活跃 Key

**新增 Edge**:
- `assigned_member` (APIKey → EnterpriseMember)
- `groups` (APIKey → Group, M:N through api_key_groups)

---

### 2.2 usage_logs 表变更

| 变更 | 字段 | 类型 | 说明 |
|:--:|------|------|------|
| 🆕 新增 | enterprise_id | int64, FK→enterprises | NULL=个人消费, 有值=企业消费 |
| 🆕 新增 | pool_type | string(20) | "personal" / "enterprise" — 审计追溯 |

**新增索引**:
- `(enterprise_id, created_at)` — 企业维度时间范围查询
- `(enterprise_id, pool_type)` — 按资金池筛选

**新增 Edge**:
- `enterprise` (UsageLog → Enterprise)

## 三、完整 ER 关系图

```
┌──────────────┐     ┌─────────────────────────┐
│  enterprises │     │ enterprise_subscriptions │ 🆕
│  id          │────→│  id                      │
│  name        │     │  enterprise_id           │
│  parent_id  ─┼→自引用│  group_id ──────→ groups │
│  status      │     │  plan_id                  │
│  balance    🆕│     │  status                   │
│  total_recharged🆕│  │  *_usage_usd             │
│  admin_user  │     └─────────────────────────┘
└──────┬───────┘
       │ 1:N
       ▼
┌──────────────────┐     ┌──────────────┐
│enterprise_members│     │ departments  │ 🆕
│  id              │     │  id          │
│  enterprise_id ◄─┼─────┤ enterprise_id│
│  user_id ────────┼→users│  parent_id   │
│  role            │     │  name        │
│  status          │     │  leader      │
│  department_id ──┼──→──┤  status      │
│  concurrency    🆕│    └──────────────┘
│  rpm_limit      🆕│
│  notes          🆕│
└──┬───────────────┘
   │ 1:N (assigned_to)
   ▼
┌──────────────┐         ┌─────────────────┐
│   api_keys   │ M:N     │ api_key_groups  │ M:1  ┌──────────┐
│  id          │────────→│  api_key_id     │─────→│  groups  │
│  user_id ────┼──→ users│  group_id       │      │  id      │
│  assigned_to─┼──┐      └─────────────────┘      │  name    │
│  usage_purp  │  │                               └──────────┘
│  bound_tool  │  │
│  name        │  │
│  status      │  │
└──────┬───────┘  │
       │          │
       │    ┌─────┘
       ▼    ▼
┌──────────────────────┐
│     usage_logs       │
│  user_id ────→ users │
│  api_key_id          │
│  enterprise_id       │────→ enterprises
│  pool_type          🆕│    "personal" | "enterprise"
│  total_cost          │
│  created_at          │
└──────────────────────┘
```

**关键关联说明**:
- `api_keys.assigned_to` → `enterprise_members.id`：分配关系
- `api_keys.user_id` → `users.id`：创建者（所有权）
- 企业 Key 消费写入 `usage_logs.enterprise_id` + `pool_type='enterprise'`
- 企业消费扣款 → `enterprises.balance`（企业余额）或 `enterprise_subscriptions`（企业套餐）
- 管理员自用 Key (assigned_to=NULL) → 扣管理员 `users.balance`，`pool_type='personal'`

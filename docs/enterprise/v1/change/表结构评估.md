## 原型功能额外需要的表/字段

### 1. `enterprises` 表 — 需增加的字段

| 新增字段 | 类型 | 原型来源 | 必要性 |
|---------|------|----------|:--:|
| `short_name` | string(100) | 创建表单 | ⭐ 必要 |
| `credit_code` | string(50) | 创建表单 统一社会信用代码 | ⭐ 必要 |
| `address` | string(500) | 创建表单 企业地址 | ⭐ 必要 |
| `scale` | string(20) | 创建表单 企业规模 | ○ 可选 |
| `industry` | string(50) | 创建表单 所属行业 | ○ 可选 |
| `description` | text | EnterpriseProfileView | ○ 可选 |
| `logo_url` | string(500) | EnterpriseProfileView | ○ 可选 |
| `notes` | text | 编辑表单 备注 | ○ 可选 |

> 🔴 **决策**: `scale` + `industry` 可做枚举，`description` + `logo_url` + `notes` 可后续迭代。核心必加: `short_name`, `credit_code`, `address`。

### 2. `enterprise_members` 表 — 需增加的字段

| 新增字段 | 类型 | 原型来源 | 必要性 |
|---------|------|----------|:--:|
| `concurrency` | int | 成员编辑 | ⭐ 必要（成员级配额） |
| `rpm_limit` | int | 成员编辑 | ⭐ 必要 |
| `department_id` | int64 FK | 成员部门归属 | ⚠️ 取决于部门表 |
| `notes` | text | 成员编辑 | ○ 可选 |

### 3. 🆕 `departments` 表 — 部门管理

**原型需要**: ✅ 有完整的部门 CRUD + 树形结构

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | int64 | PK | |
| enterprise_id | int64 | NOT NULL, FK→enterprises | 所属企业 |
| parent_id | int64 | NOT NULL, DEFAULT 0 | 父部门，0=顶级 |
| name | string(100) | NOT NULL | 部门名称 |
| order_num | int | DEFAULT 0 | 排序号 |
| leader | string(100) | | 负责人 |
| phone | string(50) | | 负责人电话 |
| email | string(255) | | 负责人邮箱 |
| status | string(20) | DEFAULT 'active' | active/disabled |
| created_at | timestamptz | | |
| updated_at | timestamptz | | |
| deleted_at | timestamptz | | 软删除 |

**复杂度评估**: 🟢 低。一张表 + 自引用树，Ent 原生支持 Edges。

### 4. 🆕 `qualifications` 表 — 企业资质文件

**原型需要**: ✅ 创建企业时上传营业执照+法人身份证

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | int64 | PK | |
| enterprise_id | int64 | NOT NULL, FK→enterprises | 所属企业 |
| type | string(20) | NOT NULL | 'business_license' / 'legal_id_card' |
| file_name | string(255) | | 原始文件名 |
| file_path | string(500) | | 存储路径 |
| file_size | int64 | | 字节 |
| status | string(20) | DEFAULT 'pending' | pending/verified/rejected |
| created_at | timestamptz | | |
| updated_at | timestamptz | | |
| deleted_at | timestamptz | | |

**复杂度评估**: 🟡 中。需要文件存储服务（本地/OSS）。

### 5. 🆕 `balance_transactions` 表 — 余额变动记录

**原型需要**: ✅ 余额充值/提现操作 + 余额历史记录

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | int64 | PK | |
| user_id | int64 | NOT NULL, FK→users | 目标用户 |
| type | string(20) | NOT NULL | 'deposit' / 'withdraw' / 'adjust' |
| amount | decimal(20,8) | NOT NULL | 金额(正=增加,负=减少) |
| balance_after | decimal(20,8) | | 操作后余额 |
| notes | text | | 备注 |
| operator_id | int64 | FK→users | 操作人 |
| created_at | timestamptz | | |

**复杂度评估**: 🟢 低。

⚠️ **但**: `users` 表已有 `balance` 字段 + `total_recharged` 字段。如果"企业余额 = 管理员余额"（设计决策3），则充值提现走现有的用户余额流程，**不需要这张表**。原型中的企业余额操作实际上操作的是管理员用户的 `users.balance`。

## 最终推荐表结构

### 核心必做 (P0)

| 表 | 操作 | 说明 |
|----|:--:|------|
| `enterprises` | 🆕 | +short_name, +credit_code, +address |
| `enterprise_members` | 🆕 | +concurrency, +rpm_limit |
| `api_key_groups` | 🆕 | 分组 M:N 中间表 |
| `api_keys` | ✏️ | +assigned_to, +usage_purpose, +bound_tool |
| `usage_logs` | ✏️ | +enterprise_id |

### 建议本期做 (P1)

| 表 | 操作 | 说明 |
|----|:--:|------|
| `departments` | 🆕 | 部门管理（含 enterprise_id） |
| `enterprise_members` | ✏️ | +department_id FK→departments |

### 本期不做 (P2+)

| 表 | 说明 |
|----|------|
| `qualifications` | 文件上传可简化为只存文件路径 |
| `balance_transactions` | 复用现有 users.balance，无需独立表 |
| enterprises 的 scale/industry/description/logo_url | JSON 扩展字段或后续迭代 |

> **核心原则**: 先跑通核心流程（企业CRUD + 成员管理 + 部门关联），附加字段按需渐进。

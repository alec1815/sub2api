## 一、企业管理页 (EnterprisesView.vue) — 平台运营方视角

**路径**: `admin/EnterprisesView.vue`

### 1.1 列表功能
- 🔍 企业名称/手机号搜索（防抖 350ms）
- 🔽 状态筛选 (active/disabled)
- 🔄 刷新按钮
- ⚙️ 列设置（可隐藏 contactName/contactPhone/rpmLimit 列，localStorage 持久化）
- 📄 分页

### 1.2 列表列
| 列 | 说明 |
|---|------|
| 企业名称 + 简称 | 带首字母头像 |
| 管理员邮箱 | adminEmail |
| 联系人 | contactName |
| 联系电话 | contactPhone |
| 余额 | 可点击查看历史，带充值入口 |
| 并发数 | concurrency |
| RPM限制 | rpm_limit |
| 状态 | active(绿)/disabled(红) |
| 创建时间 | created_at |

### 1.3 创建企业弹窗 (Create Modal)

**Section 1 - 企业基本信息**:
- 企业全称 *（必填）
- 统一社会信用代码 *（必填）
- 管理员邮箱 *（必填，格式校验）
- 企业简称
- 登录密码 *（必填，8-20位，带显隐切换）
- 确认密码 *（必填，一致性校验，带显隐切换）

**Section 2 - 联系人信息**:
- 联系人姓名 *（必填）
- 联系人手机号 *（必填，1开头11位手机号格式）
- 企业地址 *（必填）

**Section 3 - 企业分类**:
- 企业规模: micro/small/medium/large（下拉）
- 所属行业: internet/finance/manufacturing/education/healthcare/ecommerce/government/ai/software/other（下拉）

**Section 4 - 资质文件**:
- 营业执照 *（必填，pdf/jpg/jpeg/png，最大5MB）
- 法人身份证 *（必填，jpg/jpeg/png/pdf，最大5MB）

**Section 5 - 账号配置**:
- 初始余额（数字，默认0）
- 并发数（数字，默认1）
- RPM限制（数字，默认0=无限制）

### 1.4 编辑企业弹窗 (Edit Modal)
- 企业名称 *、联系人手机号 *、邮箱 *
- 密码（选填，留空不修改，带重置按钮生成随机密码）
- 用户名
- 备注
- 并发数、RPM限制
- 错误处理: DUPLICATE_NAME / DUPLICATE_PHONE / DUPLICATE_EMAIL

### 1.5 行操作按钮
| 按钮 | 功能 |
|------|------|
| 编辑 | 打开 Edit Modal |
| API Keys | 打开 EnterpriseApiKeysModal |
| 分组 | 打开 EnterpriseAllowedGroupsModal |
| 启停 | 切换 active ↔ disabled（含确认弹窗） |
| 更多... | 下拉菜单 |

### 1.6 更多菜单
- 👥 查看成员 → 跳转 EnterpriseMembersView(带 enterpriseId 参数)
- 💰 充值 → 打开 Balance Modal (add)
- 💸 提现 → 打开 Balance Modal (subtract)
- 📊 余额历史 → 打开 EnterpriseBalanceHistoryModal
- 📈 平台配额 → 打开 EnterprisePlatformQuotaModal
- 🗑️ 删除 → 确认弹窗（当前 Mock 实现）

### 1.7 余额操作弹窗
- 充值/提现切换
- 显示当前余额
- 金额 *（数字）
- 备注
- 错误处理: "Insufficient balance"

- 备注
- **部门**（下拉，从部门树构建选项）⚠️
- 并发数
- RPM限制

### 2.5 解绑确认弹窗

- 显示目标成员名称
- 错误处理: ALREADY_UNBOUND

## 五、部门管理 (departments API)

**路径**: `api/admin/departments.ts`

### 5.1 Department 类型
| 字段 | 说明 |
|------|------|
| id | PK |
| parent_id | 父部门ID，0=顶级 |
| name | 部门名称 |
| order_num | 排序号 |
| leader | 负责人 |
| phone | 负责人电话 |
| email | 负责人邮箱 |
| status | active/disabled |
| children | 子部门（树形） |

### 5.2 API 方法
- `getTree()` → 获取部门树
- `getList()` → 获取平铺列表（active状态）
- `getById(id)` → 获取单个部门
- `create(data)` → 创建部门
- `update(id, data)` → 更新部门
- `delete(id)` → 删除部门（返回hasChildren/hasMembers）

# 原型 vs 设计文档差异分析

> 对比: `need/frontend` 企业页面原型 ↔ `docs/enterprise/` 四个设计文档
> 日期: 2026-06-29

---

## 逐项对照

### A. 企业账号管理

| 功能 | 原型 | 设计文档 (04) | 差异 |
|------|:--:|:--:|------|
| 企业列表 | ✅ 搜索+过滤+列设置+分页 | ✅ Admin 端 | **一致** |
| 创建企业 | ✅ 4个Section共21个字段 | ✅ 6个字段（不含信用代码/行业/规模等） | **原型更复杂** |
| 编辑企业 | ✅ 8个字段 | ✅ name/contact_*/email | **原型多了username/notes/密码重置** |
| 启停企业 | ✅ active↔disabled | ✅ toggle + 级联禁用Key | **逻辑一致，原型不含级联可见提示** |
| 删除企业 | ✅ 更多菜单+确认弹窗 | ❌ 设计中未提及 | **原型多了删除功能** |
| 余额充值/提现 | ✅ 专用弹窗+余额历史 | ⚠️ 设计说"复用个人充值流程" | **原型独立实现** |
| 平台配额 | ✅ EnterprisePlatformQuotaModal | ❌ 未提及 | **原型独立功能** |
| 分组配置 | ✅ EnterpriseAllowedGroupsModal | ❌ 未提及(设计文档在C模块有分组M:N但非此场景) | **原型独立功能** |
| API Keys入口 | ✅ EnterpriseApiKeysModal | ❌ 未提及在企业管理页嵌入 | **原型独立功能** |

### B. 企业成员管理

| 功能 | 原型 | 设计文档 (04) | 差异 |
|------|:--:|:--:|------|
| 成员列表 | ✅ 搜索+表格 | ✅ B3 | **基本一致** |
| 代注册成员 | ✅ 邮箱/密码/用户名/部门/并发/RPM | ✅ B1 邮箱/用户名 | **原型多了部门+并发+RPM** |
| 编辑成员 | ✅ 密码/用户名/备注/部门/并发/RPM | ❌ 只有创建/绑定/解绑 | **原型多了编辑功能** |
| 解绑成员 | ✅ 确认弹窗 | ✅ B4 | **一致** |
| 绑定已有账号 | ❌ (有invite但未实现) | ✅ B2 | **设计文档有，原型未实现** |
| 部门归属 | ✅ 完整的部门树下拉 | ❌ 设计文档不含部门 | **原型多了部门管理** |

### C. 页面归属

| 页面 | 原型路径 | 设计文档路径 | 差异 |
|------|----------|-------------|------|
| 企业管理 | `admin/EnterprisesView` | Admin侧 | **一致** |
| 成员管理 | `admin/EnterpriseMembersView` | `enterprise/MemberManagement` (企业管理员视角) | **⚠️ 原型放在admin下** |
| 企业Profile | `user/EnterpriseProfileView` | 未提及独立Profile页 | **原型多了Profile页** |
| 企业财务 | ❌ 原型无 | `enterprise/EnterpriseFinance` | **设计文档有，原型无** |

## 汇总：原型比设计文档多的功能

| # | 功能 | 原型来源 | 复杂度 |
|:--:|------|----------|:--:|
| 1 | 企业删除 | EnterprisesView 更多菜单 | 低 |
| 2 | 余额充值/提现 (企业级独立入口) | Balance Modal | 中 |
| 3 | 余额历史记录 | EnterpriseBalanceHistoryModal | 中 |
| 4 | 平台配额设置 | EnterprisePlatformQuotaModal | 中 |
| 5 | 分组配置 (企业级) | EnterpriseAllowedGroupsModal | 中 |
| 6 | 资质文件上传 (营业执照/法人身份证) | Create Modal Section 4 | 中 |
| 7 | 企业分类 (规模/行业) | Create Modal Section 3 | 低 |
| 8 | 统一社会信用代码 | Create Modal | 低 |
| 9 | 联系人地址字段 | Create Modal | 低 |

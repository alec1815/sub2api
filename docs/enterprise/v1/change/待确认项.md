## Q8: API 路径 + Mock 数据

**回答**:

### API 路径
✅ 不参考原型路径，使用设计文档的 API 路由：

| 操作 | 设计文档路由 |
|------|-------------|
| 企业列表 | `GET /api/admin/enterprises` |

| 启停企业 | `POST /api/admin/enterprises/:id/toggle` |
| 成员列表 | `GET /api/enterprise/members` |
| 创建成员 | `POST /api/enterprise/members` |
| 解绑成员 | `DELETE /api/enterprise/members/:id` |

### Mock 数据
✅ **需要 Mock 数据用于前端调试**

前端开发分为两个阶段：
1. **开发阶段**: 使用前端 Mock（类似原型做法），不依赖后端
2. **联调阶段**: 切换到真实 API

**方案建议**:
- 新增 `frontend/src/api/enterprise.ts`，初期内置 Mock 数据
- 通过环境变量 `VITE_USE_MOCK=true` 切换 Mock/Real
- Mock 数据结构与设计文档 API 响应格式一致

**参考原型 Mock 做法**: 原型中的 `enterpriseAPI` 和 `enterpriseMemberAPI` 已经是完整的 Mock 实现，可直接复用其数据结构。

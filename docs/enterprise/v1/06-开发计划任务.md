## 验证门控机制

每个阶段完成后，必须通过以下门控才能进入下一阶段：

1. ✅ **编译门**：`go build ./...` 零错误
2. ✅ **生成门**：`go generate ./...` 生成代码与 schema 一致（仅 P1）
3. ✅ **测试门**：阶段相关测试全部通过
4. ✅ **对照门**：输出内容与设计文档（03/04/05）逐项对照，确认无遗漏
5. ✅ **Lint门**：零新增 lint 问题

**未通过验证绝不进入下一阶段。**

## 编码规范速查

| 层级 | 约定 |
|------|------|
| Schema | `TimeMixin` + `SoftDeleteMixin`，金额 `decimal(20,8)`，索引注解 |
| Repository | 私有结构体，实现 Service 接口，`activeQuery()`，`clientFromContext` 事务 |
| Service | 定义 Repo 接口，`infraerrors` 错误，独立 Req/Rsp |
| Handler | 读 `response.Success/Paginated`，写 `executeUserIdempotentJSON`，DTO 转换 |
| DTO | `types.go` 结构体 + `mappers.go` 映射函数 |
| Route | `Register<Module>Routes` 函数，`router.go` 统一注册 |
| Wire | `handler/wire.go` → `ProviderSet` |
| 测试 | `-tags=unit` 单测 / `-tags=integration` 集成测 |
| 提交 | 原子化、先测试通过、用户明确要求时才 commit |

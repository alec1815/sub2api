### 1.2 Mixin 决策矩阵

| Entity | SoftDeleteMixin | 原因 |
|:--|:--:|------|
| Enterprise | ✅ | 企业删除后可能恢复 |
| EnterpriseMember | ✅ | 保留解绑/离开历史 |
| Department | ✅ | 删除的部门仍有历史数据引用 |
| EnterpriseSubscription | ❌ | 生命周期由 `status` 字段管理：`active → expired → suspended` |
| APIKeyGroup | ✅ | 中间表解绑=删除 |
| APIKey | ✅ (已有) | 历史原因 |
| UsageLog | ❌ (已有) | 审计日志不删除 |

### 1.3 部分唯一索引策略

Ent 层不支持 `WHERE deleted_at IS NULL` 条件索引。策略：
- **Ent schema**: 定义普通唯一索引（供代码生成）
- **迁移 SQL**: 定义部分唯一索引（供 PGSQL 实际约束）

| 索引 | Ent schema | 迁移 SQL |
|:--|:--|:--|
| `enterprises.name` 唯一 | `Fields("name").Unique()` | `UNIQUE (name) WHERE deleted_at IS NULL` |
| `enterprise_members.user_id` 1:1 | 不定义 | `UNIQUE (user_id) WHERE status='active' AND deleted_at IS NULL` |

### 1.4 自引用字段不定义 Edge

`enterprises.parent_id` 和 `departments.parent_id` 根节点值为 0，无对应记录。如果定义 edge，Ent 要求 parent 必须存在 → 根节点无法创建。**决策：只做字段，不做 edge**。

### 2.2 分层架构约定

```
domain/            → 领域类型(Type) + 错误定义(var ErrXxx)
service/           → Repository 接口(Type Repo interface) + Service 类型别名(Type Entity = domain.Entity)
repository/        → 接口实现(私有结构体 struct + func New)
```

类型流向：`domain.Enterprise` → `service` 中 `type Enterprise = domain.Enterprise`（别名） → `repository` 中 `*service.Enterprise`

### 2.3 Ent Client 查询模式

```go
// 私有结构体
type enterpriseRepository struct { client *dbent.Client }

// activeQuery() — 统一软删除过滤
func (r *enterpriseRepository) activeQuery() *dbent.EnterpriseQuery {
    return r.client.Enterprise.Query().Where(enterprise.DeletedAtIsNil())
}

// 动态排序
func enterpriseListOrder(params) enterprise.OrderOption {
    return enterprise.ByName(sql.OrderDesc())  // 注意：OrderDesc() 是函数调用
}
```

### 2.4 Entity ↔ Service 映射模式

```go
// 单个：返回 *service.Enterprise（指针）
func enterpriseEntityToService(m *dbent.Enterprise) *service.Enterprise { ... }

// 列表：返回 []service.Enterprise（值切片）— 对齐现有代码风格
func enterpriseEntitiesToService(models []*dbent.Enterprise) []service.Enterprise { ... }
```

### 2.5 跨 Entity 查询

Repository 不限于只访问自己的 entity。如 `DepartmentRepo.HasMembers()` 直接查 `client.EnterpriseMember.Query()`：

```go
func (r *departmentRepository) HasMembers(ctx context.Context, id int64) (bool, error) {
    count, _ := r.client.EnterpriseMember.Query().
        Where(enterprisemember.DepartmentIDEQ(id),
              enterprisemember.DeletedAtIsNil()).Count(ctx)
    return count > 0, nil
}
```

### 3.3 关键业务逻辑

#### EnterpriseService 核心流程

```
CreateEnterprise(req):
  └─ 校验 name, admin_email 非空
  └─ 查找或创建管理员 users (admin_email → users 表)
  └─ 校验管理员未绑定其他企业 (memberRepo.GetByUserID)
  └─ 创建 enterprises 记录 (balance=0, total_recharged=0)
  └─ 创建 enterprise_members 记录 (role=enterprise_admin)

DeleteEnterprise(id):
  └─ 前置校验：status 必须为 disabled (否则 err)
  └─ 级联解绑所有 active 成员 (memberRepo.Unbind)
  └─ 企业套餐标记 expired (subRepo.UpdateStatus)
  └─ 软删除企业 (entRepo.SoftDelete)
```

#### EnterpriseMemberService 核心流程

```
CreateMember(entID, req):
  └─ 检查企业状态 = active
  └─ 检查 email 是否已绑定其他企业 (1:1 约束)
  └─ department_id 校验 → 属于本企业
  └─ 创建或复用 users 记录
  └─ 创建 enterprise_members 记录

UnbindMember(id):
  └─ 企业管理员不能解绑自己
  └─ status → "unbound"
  └─ 级联禁用 Key → P4 网关层处理
```

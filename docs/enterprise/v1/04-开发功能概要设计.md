**后端**:

```go
// handler/admin/enterprise.go
type CreateEnterpriseRequest struct {
    Name         string `json:"name" binding:"required,max=255"`
    ShortName    string `json:"short_name" binding:"max=100"`

    Notes        string `json:"notes"`
    ParentID     int64  `json:"parent_id"` // 0=顶级企业
}

func (h *EnterpriseHandler) CreateEnterprise(c *gin.Context) {
    // 1. 验证操作者是平台运营方 (role=admin)
    // 2. 校验企业名称不重复
    // 3. 查找或创建管理员 users 记录（若不存在则自动注册+发激活邮件）
    // 4. 创建 enterprises 记录（balance=0, total_recharged=0）
    // 5. 创建 enterprise_members 记录 (role=enterprise_admin)
    // 6. 返回企业信息
}
```

**业务规则**:
- 企业名称唯一（软删除后允许重用）
- 管理员邮箱若已被其他企业绑定为管理员 → 报错
- 管理员邮箱若已有 users 记录但非管理员 → 允许绑定
- 创建成功：企业状态 = active，balance = 0

**前端**:
- 页面：`views/admin/EnterpriseManagement.vue`
- 创建表单：企业名称（必填）、简称、信用代码、地址、规模、行业、联系人信息、管理员邮箱（必填）、管理员姓名、备注
- 支持 parent_id 选择（本期可隐藏，后续放开）

**对照原型**: 原型创建表单含 4 个 Section 共 21 个字段，本期核心字段优先，资质文件上传存文件路径即可。

## 五、模块D：计费归属与消费 🔄 V1.3 修正

### 核心规则（PRD 对齐 — 企业独立资金池）

```
assigned_to IS NULL     → 扣 api_keys.user_id 的【个人】余额/套餐
assigned_to IS NOT NULL → 扣【企业】资金池（enterprises.balance + enterprise_subscriptions）
```

> **关键修正**: 企业分配的 Key 不再扣管理员个人余额，而是扣企业独立的 `enterprises.balance` 和 `enterprise_subscriptions`。

### D1/D2/D3 — 消费归属判断

**网关层计费分流**:

```go
// gateway/billing/router.go
func DetermineBillingSource(apiKey string) (*BillingContext, error) {
    key := findAPIKey(apiKey)

    if key.AssignedTo == nil {
        // 个人 Key / 管理员自用 Key → 扣创建者个人余额/套餐
        return &BillingContext{
            OwnerUserID:        key.UserID,
            EnterpriseID:       nil,
            PayerUserID:        key.UserID,
            SubscriptionUserID: key.UserID,
            PoolType:           "personal",
        }, nil
    }

    // 企业分配的 Key → 扣企业资金池
    member := findEnterpriseMember(key.AssignedTo)
    enterprise := findEnterprise(member.EnterpriseID)

    if enterprise.Status != "active" {
        return nil, errors.New("企业已停用，调用被拒绝")
    }

    return &BillingContext{
        OwnerUserID:          key.UserID,
        EnterpriseID:         enterprise.ID,
        EnterprisePayerID:    enterprise.ID,       // 🆕 扣企业余额
        ConsumerUserID:       member.UserID,       // 实际使用者
        PoolType:             "enterprise",
    }, nil
}
```

**扣费调用**:

```go
func Consume(ctx *BillingContext, cost float64) error {
    if ctx.PoolType == "enterprise" {
        // 1. 尝试从企业套餐扣费（enterprise_subscriptions）
        // 2. 企业套餐不足 → 从企业余额扣费（enterprises.balance）
        // 3. 都不足 → 返回错误（额度不足，请联系企业管理员）
    } else {
        // 个人池（不变）：套餐 → 余额 → 报错
    }
}
```

### 三条线资金隔离（PRD D3）

| Key 类型 | created_by | assigned_to | 资金池 | 扣谁 |
|----------|:--:|:--:|:--:|------|
| 个人 Key | 任意用户 | NULL | 个人池 | 创建者的余额/套餐 |
| 管理员自用 Key | 管理员 | NULL | 管理员个人池 | 管理员的余额/套餐 |
| 企业分配 Key | 管理员 | member_id | **企业资金池** | **enterprises.balance + enterprise_subscriptions** |

**重要**: 
- 管理员个人余额 ≠ 企业余额
- 管理员有两个充值入口：「个人充值」→ users.balance, 「企业充值」→ enterprises.balance
- 管理员登录后，财务页显示两套余额

### 10.1 后端中间件

```go
func RequireEnterpriseAdmin() gin.HandlerFunc {
    return func(c *gin.Context) {
        userID := c.GetInt64("user_id")
        member := db.Where("user_id = ? AND role = ? AND status = ?",
            userID, "enterprise_admin", "active").First()
        if member == nil {
            c.AbortWithStatusJSON(403, "需要企业管理员权限")
            return
        }
        c.Set("enterprise_id", member.EnterpriseID)
        c.Set("enterprise_member_id", member.ID)
        c.Next()
    }
}
```

## 核心设计原则

- **一人一企业**：一个用户最多属于一个企业（`enterprise_members.user_id` 部分唯一索引）
- **企业独立资金池**：企业拥有独立的余额（`enterprises.balance`）和套餐（`enterprise_subscriptions`），与管理员个人资金完全隔离
- **三条线隔离**：企业 Key → 企业资金池 / 管理员自用 Key → 管理员个人池 / 成员自建 Key → 成员个人池
- **`enterprise_members` 表保留**：未来只需移除 `user_id` 唯一索引即可升级 M:N

## 四类用户状态

| # | 用户类型 | enterprise_members | 企业Key消费来源 | 个人Key消费来源 |
|:--:|----------|:--:|------|------|
| ① | 平台管理员 (users.role="admin") | 不存在 | N/A | N/A |
| ② | 个人用户 | 不存在 | N/A | 自己余额/套餐 |
| ③ | 企业管理员 | role=enterprise_admin | **企业资金池** | 自己余额/套餐 |
| ④ | 企业普通成员 | role=enterprise_member | **企业资金池** | 自己余额/套餐 |

> 详细角色体系见 [02-决策 2：两层角色体系](./v1/02-核心设计决策.md#决策-2用户角色体系)。
